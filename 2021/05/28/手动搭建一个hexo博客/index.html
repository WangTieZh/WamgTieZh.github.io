手动搭建一个hexo博客

Hexo是一款基于Nodejs的，快速、简洁且高效的博客框架。具有丰富的插件和主题，具有超快的速度。支持Makedown语法，可以方便快捷的编写博客文档。同时支持node命令，可以一键部署到GitHub Pages, Heroku 或其他平台，总之，是个写博客的利器。

使用hexo前，首先要保证自己的电脑已经安装了Git 和 Node。且Node版本不低于10.13。

使用npm或者yarn安装hexo的脚手架工具，这里以npm为例
npm install -g hexo-cli

创建一个博客项目
hexo init hexo-demo

进入hexo-demo文件夹，安装依赖

cd hexo-demo
npm install

安装完依赖以后，执行以下命令，启动hexo服务器，运行结果如图。
hexo server

可在github或者gitee上建站，gitee国内访问快一些

1、新建一个仓库

格式必须是：用户名+.gitee.io 只有这样，将来要部署到Gite e page的时候，才会被识别，也就是xxxx.gitee.io，其中xxx就是你注册Gitee的用户名。

这里还需要安装一个插件，它是hexo的上传插件 deploy-git

将hexo博客站点上传到gitee上

然后我们打开hexo的配置文件 config.yml

修改repo 为你仓库的地址。

之后就可以推送博客站点到gitee上了

推送命令

hexo d 就可推送到gitee上了。

然后就可以申请，gitee-pages服务，当我们提交后，会需要两天的审核期。

然后要修改配置文件的url 就是后续gitee pages审核成功之后的地址。

如果要切换主题的话，可以在配置文件中修改，主题下载到项目中的themes，配置文件和就会生效

gitee上传新文件之后，需要手动更新一下，更新后的页面才会生效
